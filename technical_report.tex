\section*{Summary}
\addcontentsline{toc}{section}{Summary}
\sectionrule

\code{nft-marketplace-v2} is a local fixed-price NFT marketplace. It has an ERC-721 minting contract, a marketplace contract that holds listed NFTs in escrow, scripts for deployment and demo runs, tests, and a React frontend that works through Viem.

The project focuses on one clear flow: mint an NFT, approve the marketplace, list it for ETH, buy it from another account, and settle the sale with a marketplace fee. The code is built for local development first, with enough structure to move toward a public testnet later.

\section{Project Scope}

The marketplace is intentionally narrow. It does not try to cover every feature found in large NFT platforms. The useful parts are implemented and tested:

\begin{itemize}
    \item ERC-721 minting with token URI storage.
    \item Escrowed fixed-price listings.
    \item Listing price updates and cancellations by the seller.
    \item Exact-price purchases with seller and fee-recipient payouts.
    \item Local deployment, demo, test, and frontend workflows.
\end{itemize}

\section{Contract Design}

\subsection{MyNFT}

\code{MyNFT} extends OpenZeppelin \code{ERC721URIStorage}. It names the collection \textit{Viem Marketplace NFT}, uses the symbol \textit{VMNFT}, and increments \code{totalMinted} for every new token.

The contract has two minting functions:

\begin{itemize}
    \item \code{mint(string tokenURI)} mints to the caller.
    \item \code{mintTo(address to, string tokenURI)} mints to a chosen address.
\end{itemize}

Minting to the zero address is rejected with \code{InvalidRecipient}.

\subsection{NFTMarketplace}

\code{NFTMarketplace} stores each listing in a \code{Listing} struct. A second mapping, \code{listingByToken}, points from an NFT contract and token ID to the active listing ID. This prevents the frontend from guessing which listing belongs to a token.

\begin{lstlisting}[style=contractstyle, language=Java, caption={Listing state stored by the marketplace.}]
enum ListingStatus {
    None,
    Active,
    Sold,
    Canceled
}

struct Listing {
    uint256 listingId;
    address nft;
    uint256 tokenId;
    address payable seller;
    uint256 price;
    ListingStatus status;
    address buyer;
}
\end{lstlisting}

The contract receives ERC-721 tokens through \code{IERC721Receiver}, limits fee changes through \code{Ownable}, and protects listing and purchase flows with \code{ReentrancyGuard}.

\subsection{Listing Flow}

\begin{enumerate}
    \item The seller owns an NFT.
    \item The seller approves the marketplace for that token.
    \item The seller calls \code{createListing}.
    \item The marketplace records the listing and transfers the NFT into escrow.
    \item The seller can update the price or cancel while the listing is active.
    \item A buyer calls \code{purchaseListing} and sends exactly the listed price.
    \item The marketplace transfers the NFT to the buyer and pays ETH to the seller and fee recipient.
\end{enumerate}

\subsection{Fee Rules}

Fees use basis points. The denominator is \code{10,000}, so \code{250} basis points is \code{2.5\%}. The maximum fee is capped at \code{1,000} basis points, or \code{10\%}, through \code{MAX\_MARKETPLACE\_FEE\_BPS}.

\[
    \text{marketplace fee} = \frac{\text{price} \times \text{fee bps}}{10000}
\]

\[
    \text{seller proceeds} = \text{price} - \text{marketplace fee}
\]

\section{Frontend}

The frontend in \code{src/App.tsx} uses a public Viem client for reads and an injected wallet client for writes. It supports wallet connection, chain switching, NFT minting, approval and listing, market browsing, buying, and seller-side cancellation.

Metadata is read from \code{tokenURI}. The app handles direct image URLs, JSON metadata URLs, and \code{ipfs://} links by converting them to an IPFS gateway URL. If metadata cannot be loaded, the UI falls back to a simple token label.

The market screen includes search, filters, sorting, listing cards, active listing counts, floor price, total ask value, and the connected user's owned or listed items.

\section{Local Workflow}

The normal local run is:

\begin{enumerate}
    \item Install dependencies with \code{npm install}.
    \item Start the local chain with \code{npm run node}.
    \item Deploy contracts with \code{npm run deploy}.
    \item Start the frontend with \code{npm run dev}.
    \item Connect a wallet to chain \code{31337}.
\end{enumerate}

The deploy script deploys \code{MyNFT}, deploys \code{NFTMarketplace} with the fee recipient and a default fee of \code{250} basis points, then writes the contract addresses to \code{src/deployment.ts}.

\section{Testing}

The test file \code{test/NFTMarketplace.ts} checks the main contract behavior:

\begin{itemize}
    \item minting and token URI storage;
    \item escrowed listing creation;
    \item seller-only price updates and cancellations;
    \item purchase settlement, NFT transfer, seller proceeds, and fee payment;
    \item rejection of invalid prices, unauthorized actions, self-purchases, and incorrect ETH amounts;
    \item owner-only fee and fee-recipient changes.
\end{itemize}

\section{Security Notes}

The project uses several basic safeguards:

\begin{itemize}
    \item Listed NFTs are held in marketplace escrow.
    \item Buyers must pay exactly the listing price.
    \item Listing creation, cancellation, and purchase use \code{nonReentrant}.
    \item Purchase state is updated before NFT and ETH transfers.
    \item Custom errors make failed actions easier to trace.
    \item The marketplace fee cannot be set above \code{10\%}.
\end{itemize}

The current version still has clear limits. It supports only fixed-price sales, scans listing IDs from \code{1} to \code{listingCount}, allows open minting, has no royalty standard such as ERC-2981, and sends ETH with direct calls that revert the purchase if a payout fails.

\section{Next Steps}

The most useful improvements would be:

\begin{itemize}
    \item event-based indexing for listing discovery;
    \item pagination and automatic market refresh;
    \item ERC-2981 royalty support;
    \item bid or auction listings;
    \item clearer multi-network deployment settings;
    \item static analysis and contract coverage reports.
\end{itemize}

\section{Conclusion}

This project builds a working local NFT marketplace around a small set of contract rules. The contracts handle minting, escrow, listing status, fee calculation, purchases, and seller payouts. The frontend gives a browser-based way to use those flows through a connected wallet. The code is suitable for demonstration, testing, and further extension toward indexing, royalties, and public testnet deployment.
